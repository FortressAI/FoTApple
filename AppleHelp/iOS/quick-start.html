For Clinicians

FoT Clinician - Medical Decision Support

What You'll Need: - iPhone (iOS 17+), Mac (macOS 14+), or Apple Watch (watchOS 10+) - 5 minutes to set up - Internet connection (for initial setup and drug database)

Step 1: Install (2 minutes)

TestFlight Beta (Available Now): 1. Email beta@fortressai.com with your: - Medical specialty - NPI number (for verification) - Device preference (iPhone/Mac/Watch) 2. Receive TestFlight invitation within 24-48 hours 3. Install app from TestFlight link

App Store (Coming Q2 2026): - Search "FoT Clinician" - Download and install - 30-day free trial included

Step 2: Set Up Your Account (1 minute)

  1. Launch FoT Clinician
  2. Create account with Face ID or Touch ID
  3. Configure preferences:
  4. Units (metric or imperial)
  5. Clinical guidelines (ACC/AHA, GINA, etc.)
  6. Notification settings
  7. Complete brief tutorial (skip if experienced)

Step 3: Try Key Features (2 minutes)

Add Your First Patient:

1. Tap "+" button
2. Enter patient details:
   - Name (or use "John Doe" for demo)
   - Age: 68
   - Gender: Male
3. Add allergies: Penicillin
4. Add current medications: 
   - Metformin 1000mg PO BID
   - Lisinopril 20mg PO daily
5. Save patient

Check Drug Interactions:

1. Select patient "John Doe"
2. Tap "Encounter" tab
3. Scroll to "Medications" section
4. Tap "Add Medication"
5. Search "aspirin"
6. Select "Aspirin 81mg"
7. Tap "Check Interactions"
8. ⚠️ See interaction warning with metformin
9. Review mechanism and recommendations

Generate SOAP Note:

1. In encounter, add:
   - Chief Complaint: "Chest pain"
   - Vitals: BP 145/92, HR 98, RR 18
2. Scroll to "SOAP Note" section
3. Tap "Generate"
4. Review auto-generated professional note
5. Tap "Export" to share (PDF)

View Cryptographic Proof:

1. At bottom of SOAP note
2. See "Blockchain Proof: 01JD7X8K..."
3. Tap to view complete proof
4. See Merkle root, Ed25519 signature
5. Independently verifiable on SafeAICoin

What You'll Need: - iPhone (iOS 17+), iPad (iOS 17+), or Mac (macOS 14+) - 5 minutes to set up - Your bar admission information

  1. Launch FoT Legal US

  6. Court rules (FRCP, local rules)

Step 3: Try Key Features (2 minutes)

Create Your First Case:

1. Tap "+" button
2. Enter case details:
   - Name: Smith v. Johnson Corp.
   - Type: Civil - Breach of Contract
   - Court: US District Court, ND California
   - Filed: [Today's date]
3. Add parties:
   - Client: Sarah Smith
   - Opposing: Johnson Corporation
4. Save case

Automatic Deadline Calculation:

1. Open case "Smith v. Johnson"
2. View "Deadlines" tab
3. See automatically calculated deadlines:
   - Answer Due: 21 days (FRCP 12(a)(1)(A)(i))
   - Initial Disclosures: [Date] (FRCP 26(a)(1)(C))
   - Discovery Conference: [Date] (FRCP 26(f))
4. Each deadline includes rule citation
5. Reminders automatically set

Generate Bluebook Citation:

1. Tap "Research" tab
2. Search "NFIB v Sebelius"
3. Select case from results
4. Tap "Copy Citation"
5. Citation copied in Bluebook format:
   Nat'l Fed'n of Indep. Bus. v. Sebelius, 567 U.S. 519 (2012).

Constitutional Analysis:

1. Tap "Analysis" tab
2. Enter question: "Can Congress require health insurance?"
3. System provides:
   - Relevant constitutional provisions
   - Supreme Court precedent (NFIB v. Sebelius)
   - Analysis of Commerce Clause vs. Taxing Power
   - Recommended strategy
4. Review blockchain proof of analysis

For Developers

Build and Contribute to FoT Apple

What You'll Need: - Mac with macOS 14+ (Sonoma or later) - Xcode 15+ - Swift 5.9+ - 15 minutes for first build

Step 1: Clone Repository (2 minutes)

# Clone the repository
git clone https://github.com/FortressAI/FoTApple.git
cd FoTApple

# Repository structure
FoTApple/
├── packages/          # Swift packages (6 packages)
├── apps/             # Platform apps (iOS, macOS, watchOS)
├── Sources/          # Core source code
├── Tests/            # Test suites
└── Package.swift     # Main package manifest

Step 2: Build All Packages (5 minutes)

# Resolve dependencies
swift package resolve

# Build all packages
swift build

# Expected output:
Build complete! (45 seconds)

Verify Package Build:

# Build individual packages
cd packages/FoTCore && swift build      # ✅ Success
cd ../RulesEngine && swift build        # ✅ Success
cd ../DataAdapters && swift build       # ✅ Success
cd ../FoTClinician && swift build       # ✅ Success
cd ../FoTLegalUS && swift build         # ✅ Success
cd ../FoTEducationK18 && swift build    # ✅ Success

Step 3: Run Tests (3 minutes)

# Run all tests
cd /path/to/FoTApple
swift test

# Expected output:

     Executed 54 tests, with 0 failures (0 unexpected)

Test Breakdown: - FoTCore: 8 tests ✅ - RulesEngine: Tests ✅ - DataAdapters: 15+ tests ✅ - FoTClinician: 31 tests ✅ - FoTLegalUS: Tests ✅ - FoTEducationK18: Tests ✅

Step 4: Build iOS App (5 minutes)

cd apps/ClinicianApp/iOS

Common Questions

Can I use FoT Apple offline?

Yes! Core features work offline: - ✅ Patient/case/student management (all apps) - ✅ SOAP note generation (Clinician) - ✅ Deadline calculations (Legal US) - ✅ Assignment tracking (Education K-18) - ⚠️ Drug interaction checking requires internet (initial download, then offline) - ⚠️ Blockchain attestation requires internet (queued for sync)

Is my data secure?

Absolutely. - All data encrypted with AES-256-GCM - Encryption keys stored in Secure Enclave (hardware-backed) - Biometric authentication (Face ID, Touch ID) - HIPAA-compliant architecture (medical) - FERPA/COPPA-compliant (education) - Attorney-client privilege protection (legal) - Local processing (data doesn't leave your device unless you export)
