Pour les Cliniciens

FoT Clinician - Aide à la Décision Médicale

Ce dont vous aurez besoin : - iPhone (iOS 17+), Mac (macOS 14+), ou Apple Watch (watchOS 10+) - 5 minutes pour la configuration - Connexion Internet (pour la configuration initiale et la base de données des médicaments)

Étape 1 : Installation (2 minutes)

Bêta TestFlight (Disponible Maintenant) : 1. Envoyez un e-mail à beta@fortressai.com avec : - Votre spécialité médicale - Numéro NPI (pour vérification) - Préférence d'appareil (iPhone/Mac/Watch) 2. Recevez l'invitation TestFlight sous 24-48 heures 3. Installez l'application depuis le lien TestFlight

App Store (Prévu T2 2026) : - Recherchez "FoT Clinician" - Téléchargez et installez - Essai gratuit de 30 jours inclus

Étape 2 : Configurez Votre Compte (1 minute)

  1. Lancez FoT Clinician
  2. Créez un compte avec Face ID ou Touch ID
  3. Configurez les préférences :
  4. Unités (métriques ou impériales)
  5. Directives cliniques (ACC/AHA, GINA, etc.)
  6. Paramètres de notification
  7. Complétez le bref tutoriel (passez si expérimenté)

Étape 3 : Essayez les Fonctions Clés (2 minutes)

Ajoutez Votre Premier Patient :

1. Appuyez sur le bouton "+"
2. Entrez les détails du patient :
   - Nom (ou utilisez "Jean Dupont" pour la démo)
   - Âge : 68
   - Genre : Masculin
3. Ajoutez les allergies : Pénicilline
4. Ajoutez les médicaments actuels :
   - Metformine 1000mg PO BID
   - Lisinopril 20mg PO quotidien
5. Sauvegardez le patient

Vérifiez les Interactions Médicamenteuses :

1. Sélectionnez le patient "Jean Dupont"
2. Appuyez sur l'onglet "Consultation"
3. Faites défiler jusqu'à la section "Médicaments"
4. Appuyez sur "Ajouter Médicament"
5. Recherchez "aspirine"
6. Sélectionnez "Aspirine 81mg"
7. Appuyez sur "Vérifier Interactions"
8. ⚠️ Voyez l'avertissement d'interaction avec la metformine
9. Examinez le mécanisme et les recommandations

Générez une Note SOAP :

1. Dans la consultation, ajoutez :
   - Motif de consultation : "Douleur thoracique"
   - Signes vitaux : TA 145/92, FC 98, FR 18
2. Faites défiler jusqu'à la section "Note SOAP"
3. Appuyez sur "Générer"
4. Examinez la note professionnelle auto-générée
5. Appuyez sur "Exporter" pour partager (PDF)

Visualisez la Preuve Cryptographique :

1. En bas de la note SOAP
2. Voyez "Preuve Blockchain : 01JD7X8K..."
3. Appuyez pour voir la preuve complète
4. Voyez la racine Merkle, signature Ed25519
5. Vérifiable indépendamment sur SafeAICoin

Pour les Avocats

FoT Legal US - Gestion de Cabinet Juridique

Ce dont vous aurez besoin : - iPhone (iOS 17+), iPad (iOS 17+), ou Mac (macOS 14+) - 5 minutes pour la configuration - Vos informations d'admission au barreau

Étape 1 : Installation (2 minutes)

Bêta TestFlight (Disponible Maintenant) : 1. Envoyez un e-mail à beta@fortressai.com avec : - Admission au barreau de l'État et année - Numéro d'inscription au barreau (pour vérification) - Domaines de pratique 2. Recevez l'invitation TestFlight 3. Installez l'application

Étape 2 : Configurez Votre Compte (1 minute)

  1. Lancez FoT Legal US
  2. Créez un compte avec Face ID ou Touch ID
  3. Configurez les préférences :
  4. Juridiction principale (fédérale/d'État)
  5. Domaines de pratique (civil, pénal, etc.)
  6. Règles du tribunal (FRCP, règles locales)
  7. Complétez le tutoriel

Étape 3 : Essayez les Fonctions Clés (2 minutes)

Créez Votre Premier Dossier :

1. Appuyez sur le bouton "+"
2. Entrez les détails du dossier :
   - Nom : Smith c. Johnson Corp.
   - Type : Civil - Rupture de contrat
   - Tribunal : Cour de district américaine, ND Californie
   - Déposé : [Date du jour]
3. Ajoutez les parties :
   - Client : Sarah Smith
   - Partie adverse : Johnson Corporation
4. Sauvegardez le dossier

Calcul Automatique des Délais :

1. Ouvrez le dossier "Smith c. Johnson"
2. Consultez l'onglet "Délais"
3. Voyez les délais automatiquement calculés :
   - Réponse due : 21 jours (FRCP 12(a)(1)(A)(i))
   - Communications initiales : [Date] (FRCP 26(a)(1)(C))
   - Conférence de découverte : [Date] (FRCP 26(f))
4. Chaque délai inclut la référence de règle
5. Rappels automatiquement définis

Générez une Citation Bluebook :

1. Appuyez sur l'onglet "Recherche"
2. Recherchez "NFIB v Sebelius"
3. Sélectionnez l'affaire dans les résultats
4. Appuyez sur "Copier Citation"
5. Citation copiée au format Bluebook :
   Nat'l Fed'n of Indep. Bus. v. Sebelius, 567 U.S. 519 (2012).

Analyse Constitutionnelle :

1. Appuyez sur l'onglet "Analyse"
2. Entrez la question : "Le Congrès peut-il exiger une assurance santé ?"
3. Le système fournit :
   - Dispositions constitutionnelles pertinentes
   - Précédent de la Cour suprême (NFIB v. Sebelius)
   - Analyse Commerce Clause vs. Pouvoir fiscal
   - Stratégie recommandée
4. Examinez la preuve blockchain de l'analyse

Pour les Développeurs

Construire et Contribuer

Ce dont vous aurez besoin : - Mac avec macOS 14+ (Sonoma) - Xcode 15.0+ - Git - 10 minutes pour la configuration initiale

Étape 1 : Cloner le Dépôt (1 minute)

# Cloner le dépôt principal
git clone https://github.com/FortressAI/FoTApple.git
cd FoTApple

# Voir la structure
ls -la

Structure du Dépôt :

FoTApple/
├── apps/                    # 3 applications (Clinician, Legal, Education)
├── packages/                # 6 packages Swift réutilisables
├── scripts/                 # Scripts d'automatisation
└── docs/                    # Documentation

Étape 2 : Configuration Automatisée (5 minutes)

Pour l'Application Clinician iOS :

cd apps/ClinicianApp/iOS
./setup.sh

Ce que fait setup.sh : 1. ✅ Vérifie les dépendances système (Xcode, xcodegen) 2. ✅ Installe xcodegen si manquant 3. ✅ Génère le fichier projet Xcode 4. ✅ Résout les dépendances des packages Swift 5. ✅ Crée la configuration initiale 6. ✅ Exécute la validation de l'infrastructure

Durée : ~5 minutes lors de la première exécution

Étape 3 : Construire et Tester (4 minutes)

Ouvrir dans Xcode :

# Après setup.sh
open ClinicianApp.xcodeproj

Ou Utiliser le Script de Construction :

# Exécuter les tests
./build.sh test

# Construire pour iPhone
./build.sh run

# Construire pour Mac
./build.sh run-mac

# Voir toutes les options
./build.sh help

Premiers Tests :

# Exécuter la suite complète (54+ tests)
./build.sh test

# Exécuter des tests spécifiques
xcodebuild test \
  -scheme ClinicianApp \
  -destination 'platform=iOS Simulator,name=iPhone 15'
