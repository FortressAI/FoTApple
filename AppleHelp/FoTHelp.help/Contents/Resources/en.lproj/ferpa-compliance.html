FERPA Overview

What is FERPA?

The Family Educational Rights and Privacy Act (20 U.S.C. § 1232g; 34 CFR Part 99) is a Federal law that protects the privacy of student education records. The law applies to all schools that receive funds under an applicable program of the U.S. Department of Education.

Key Rights Under FERPA:

  1. Right to Inspect: Parents have the right to inspect and review their child's education records maintained by the school
  2. Right to Amend: Parents have the right to request that a school correct records believed to be inaccurate or misleading
  3. Right to Consent: Generally, schools must have written permission from parents to release any information from a student's education record
  4. Right to File Complaint: Parents have the right to file a complaint with the U.S. Department of Education

Age Threshold: When a student turns 18 or attends a postsecondary institution, these rights transfer to the student.

COPPA Overview

What is COPPA?

The Children's Online Privacy Protection Act (15 U.S.C. §§ 6501-6506) requires operators of websites or online services directed to children under 13 to obtain verifiable parental consent before collecting, using, or disclosing personal information from children.

COPPA Requirements:

  1. Parental Consent: Must obtain verifiable parental consent before collecting personal information
  2. Privacy Notice: Must provide clear notice of data collection practices
  3. Parental Control: Parents must be able to review and delete their child's information
  4. Data Security: Must maintain reasonable security procedures
  5. Data Retention: Must not retain data longer than necessary

4. Access Control Implementation

Principle of Least Privilege:

Users only have access to data necessary for their legitimate educational interest.

Role-Based Access Control (RBAC):

Role Access Permissions
Student Own records only (grades, assignments, feedback)
Parent/Guardian Own child's records only (full educational record)
Teacher Students in their classes only (create, view, grade)
School Administrator All students in their school (oversight, reporting)
District Administrator Aggregated data only (de-identified analytics)
System Administrator Technical access only (no viewing educational content)

Technical Implementation:

// Access control check before every data operation
func checkAccess(user: User, student: Student, operation: Operation) throws {
    switch user.role {
    case .student:
        guard user.id == student.id else {
            throw FERPAError.unauthorizedAccess
        }

    case .parent:
        guard user.children.contains(student.id) else {
            throw FERPAError.unauthorizedAccess
        }

    case .teacher:
        guard user.classes.contains(where: { $0.students.contains(student.id) }) else {
            throw FERPAError.unauthorizedAccess
        }

    case .schoolAdmin:
        guard user.school == student.school else {
            throw FERPAError.unauthorizedAccess
        }

    case .districtAdmin:
        // Can only access aggregated, de-identified data
        guard operation == .viewAggregatedData else {
            throw FERPAError.unauthorizedAccess
        }
    }

    // Log access for audit trail
    await auditLog.record(
        user: user.id,
        student: student.id,
        operation: operation,
        timestamp: Date(),
        blockchainProof: generateProof()
    )
}

COPPA Compliance (Students Under 13)

Verifiable Parental Consent

COPPA requires verifiable parental consent before collecting personal information from children under 13.

FoT Education K-18 Consent Process:

For School-Based Use (School Authorization):

Under COPPA, schools can provide consent on behalf of parents for educational purposes (the "school official" exception). However, FoT Education K-18 implements additional parental consent as best practice.

Parental Consent Workflow:

ENROLLMENT PROCESS (Student Under 13):

Step 1: Teacher/Admin creates student account
        → System detects age < 13
        → Account created in "Pending Consent" status
        → No personal data collection yet

Step 2: System generates unique consent link
        → Sent to parent email
        → Link expires in 30 days
        → Reminder emails at 7, 14, 21 days

Step 3: Parent accesses consent form
        → Clear disclosure of data collection practices
        → Plain language explanation (8th grade reading level)
        → List of data types collected
        → How data will be used
        → Retention policies
        → Parent rights

Step 4: Parent provides consent
        → Digital signature (typed name + checkbox)
        → Identity verification (last 4 of SSN OR answer security questions)
        → Timestamp and IP address logged
        → Consent stored with blockchain attestation

Step 5: Account activated
        → Student can now use FoT Education K-18
        → Parent receives confirmation email
        → Parent Portal access granted

CONSENT REVOCATION:
Parent can revoke consent at any time:
→ Account immediately disabled
→ No further data collection
→ Existing data retained per FERPA (educational records)
   OR deleted per parent request (within 30 days)

Automated Deletion:

// Automated deletion after retention period
func enforceRetentionPolicy() async {
    let retiredStudents = await getStudentsRetired()

    for student in retiredStudents {
        let graduationDate = student.graduationDate
        let retentionPeriod = student.state.retentionRequirement // 7 years
        let deletionDate = graduationDate + retentionPeriod

        if Date() >= deletionDate {
            // Delete non-essential records
            await deleteAssignments(student.id)
            await deleteTemporaryNotes(student.id)

            // Archive essential records (encrypted)
            await archiveTranscript(student.id)
            await archiveIEP(student.id)

            // Log deletion
            await auditLog.record(
                operation: .autoDelete,
                student: student.id,
                date: Date(),
                reason: "Retention period expired",
                blockchainProof: generateProof()
            )
        }
    }
}
