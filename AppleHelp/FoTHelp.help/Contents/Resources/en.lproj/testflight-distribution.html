üë• Inviting Testers

Internal Testers (Up to 100)

Who: Your team members with App Store Connect roles

How: 1. Go to TestFlight ‚Üí Internal Testing 2. Click Add Testers 3. Select team members 4. ‚úÖ They can install immediately (no review needed)

Benefits: - No Apple review required - Install builds immediately - Test before external release - Up to 100 devices per internal tester

External Testers (Up to 10,000)

Option A: Email Invites

How: 1. Go to TestFlight ‚Üí External Testing 2. Click Add Group (e.g., "Beta Testers") 3. Click Add Testers 4. Enter email addresses (one per line) 5. They receive email with TestFlight link

Benefits: - Controlled invite list - Track who accepted - Send updates to specific groups

Option B: Public Link (Recommended for Mass Testing!)

How: 1. Go to TestFlight ‚Üí External Testing ‚Üí Public Link 2. Enable link and copy URL 3. Share on: - Twitter/X: "Join the Personal Health beta! [link]" - Product Hunt beta list - Reddit: r/iosapps, r/mentalhealth - Your website - Email campaigns

Set criteria (optional): - Device type (iPhone only, iPad, Mac, etc.) - OS version (iOS 17+, macOS 14+, etc.) - Maximum testers (10,000 limit)

Benefits: - Don't need email addresses - Easy to share widely - Can track views vs. installs - Can disable when full - Perfect for going viral

Example Public Link:

https://testflight.apple.com/join/ABC123XYZ

üîß Troubleshooting

"No matching provisioning profiles found"

Solution: Let Xcode handle this automatically

# In Xcode: Signing & Capabilities ‚Üí Automatically manage signing ‚úÖ

Or manually: Developer Portal ‚Üí Certificates, IDs & Profiles ‚Üí Profiles

"AuthKey permission denied"

Solution:

chmod 600 .apple-keys/AuthKey_43BGN9JC5B.p8

"Upload failed: authentication error"

Check: - Issuer ID is correct - Auth Key path is correct - Key ID is 43BGN9JC5B

"App not found in App Store Connect"

Solution: Create app entry first (Step 2 above)

Verify: - Bundle ID matches exactly - App is in "Prepare for Submission" state

"Processing failed"

Common causes: - Missing Info.plist keys - Invalid bundle identifier - Missing app icons - Privacy usage descriptions missing

Check: - NSHealthShareUsageDescription - NSHealthUpdateUsageDescription - NSCameraUsageDescription - NSPhotoLibraryUsageDescription - NSLocationWhenInUseUsageDescription

üí° Pro Tips

1. Start with Internal Testing

  • Test with your team first
  • Fix obvious bugs before external release
  • Internal testers can install immediately (no review delay)

2. Use Public Link for Beta Launch

  • Share on Twitter/X: "Join the Personal Health beta! [link]"
  • Can go viral if useful
  • No email collection needed
  • Easy to promote

3. Monitor Feedback Closely

  • Check TestFlight feedback daily
  • Respond to crash reports within 24 hours
  • Thank testers publicly on social media
  • Feature helpful testers in release notes

4. Iterate Fast

  • Upload new builds frequently (1-2 per week)
  • Each build goes to existing testers automatically (if enabled)
  • Show you're actively improving
  • Keep testers engaged

5. Track Metrics

  • App Store Connect shows:
  • Invited vs. installed conversion
  • Session counts and duration
  • Crash rates by device/OS
  • Feedback volume and sentiment
  • Use data to prioritize fixes

üéâ After TestFlight: App Store Submission

Once you've tested and refined:

1. App Store Metadata

Create in App Store Connect: - Name: Personal Health (or FoT Clinician) - Subtitle: "Complete Mind & Body Health Monitor" - Description: Full app description (4000 chars) - Keywords: health, mental health, tracking, cryptographic, receipts, wellness, mood, journal - Screenshots: 5-10 per device type - App Preview Video: 30-second demo

2. Submit for Review

  • Navigate to: App Store Connect ‚Üí Your App ‚Üí App Store
  • Add all required metadata
  • Submit for review
  • Answer reviewer questions promptly

3. Apple Review Timeline

  • 1-3 days typically for standard apps
  • May be longer for health apps (medical review)
  • Fix any issues they find quickly
  • Resubmit if rejected

4. Release

  • Manual release: Hit "Release" when approved
  • Automatic release: Goes live immediately upon approval
  • Scheduled release: Pick a date/time
