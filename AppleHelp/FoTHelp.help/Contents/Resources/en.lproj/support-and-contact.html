When I try to check drug interactions for a patient on warfarin + aspirin, the search spinner shows but never completes. This has happened 3 times today.

1. Open patient "John Doe"
2. Tap Encounter tab
3. Tap Medications section
4. Tap "Add Medication"
5. Search "aspirin"
6. Spinner appears but never stops

I have good WiFi connection. Other parts of the app work fine.

Screenshot attached.

  4. GitHub Issues - Search for similar problems

Common Issues & Solutions

"Drug interaction search not working" - Check internet connection (requires RxNav API) - Verify drug database downloaded (Settings → Drug Database → Download) - Try restarting app - Check RxNav API status: status.nlm.nih.gov

"Can't log in with Face ID" - Verify Face ID enabled in iOS Settings - Try password login as alternative - Re-enroll Face ID: Settings → Face ID & Passcode - Contact support if persists

"App crashes on launch" - Force quit app (swipe up in app switcher) - Restart device - Check for iOS/macOS updates - Reinstall app - Contact support with crash logs

"Data not syncing between devices" - Verify iCloud enabled (Settings → [Your Name] → iCloud) - Check internet connection - Verify signed in to same Apple ID on all devices - Force sync: Settings → Sync → Sync Now - Note: iCloud sync coming Q1 2026, currently manual export/import

"SOAP note won't export" - Check storage space on device - Try different export format (PDF vs. text) - Restart app - Contact support if persists
